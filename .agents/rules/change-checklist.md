# Change Checklist

适用于提交前或宣称改动完成前的联动自查。

## API 改动

- 是否同步更新前端 `src/api/*`
- 是否同步更新前端 `src/types/*`
- 是否需要更新 `README` 或 `docs/` 中的接口说明
- 是否误用了已废弃接口模式

## 设置与配置改动

- 是否同步更新 `internal/entity/setting.go`
- 是否同步更新 service 读取逻辑
- 是否同步更新后台设置页与前端类型
- 是否检查了敏感值加密存储和脱敏返回

## 内容与社交功能改动

- 是否检查了点赞数、收藏数、评论数、关注数等计数一致性
- 是否检查了列表分页、排序、游标返回
- 是否检查了权限和可见性边界

## 上传与资源访问改动

- 是否检查 presign 上传、返回 key、访问 URL 转换
- 是否检查附件和头像两条链路
- 是否检查了 S3 public URL / presigned GET 行为

## 前端改动

- 是否同步更新页面、API、类型、路由
- 是否破坏启动流程、登录流程、初始化流程
- 是否保持现有布局与导航结构

## 验证

- 是否运行了与改动匹配的验证命令
- 是否明确说明了未验证部分
