# 内容审核机制验收 Checklist

## 状态

本文档用于验收当前已经实现的内容审核与可见性机制，重点关注：

- 现有主路径是否顺畅
- 权限与可见性是否一致
- 后台审核体验是否达到预期
- 已实现功能是否存在明显回归

本文档不是 roadmap，也不是新增功能设计稿，而是一份手工验收清单。

## 使用方式

建议按以下顺序执行：

1. 先准备两个账号
   - 一个普通作者账号
   - 一个管理员账号
2. 准备至少 3 条测试内容
   - 一条草稿
   - 一条待审核内容
   - 一条已审核公开内容
3. 每完成一项，在本地记录结果
   - `通过`
   - `失败`
   - `需优化体验`
4. 若发现问题，优先记录：
   - 复现步骤
   - 预期结果
   - 实际结果

## 验收目标

本轮验收重点确认：

- 作者提交流程是否清楚
- 管理员审核流程是否顺手
- 公共可见性规则是否真正收口
- 已实现交互是否存在明显困惑点

## 一、作者路径

### 1. 草稿创建与保存

- [ ] 作者可以创建一条新内容
- [ ] 新内容默认进入 `草稿`
- [ ] 草稿不会出现在公共列表
- [ ] 草稿详情只有作者和管理员可见

### 2. 提交审核

- [ ] 作者可以从草稿执行“提交审核”
- [ ] 提交后内容状态变为 `已发布 / 待审核 / 私有`
- [ ] “我的内容”中可以看到待审核状态
- [ ] 页面提示能明确说明“审核通过前不会出现在公开列表中”

### 3. 已发布内容再次编辑

- [ ] 作者编辑已公开内容后，主按钮文案清楚表达“重新进入审核”
- [ ] 保存后内容重新进入 `已发布 / 待审核 / 私有`
- [ ] 内容会暂时从公共列表移除
- [ ] 作者仍可继续查看和编辑

### 4. 撤回为草稿

- [ ] 作者可以把已发布内容撤回为草稿
- [ ] 撤回前有确认提示
- [ ] 撤回后内容从已发布列表消失
- [ ] 在草稿筛选下能找到该内容

### 5. 审核反馈感知

- [ ] 作者能在“我的内容”中看到审核状态
- [ ] 作者能在“我的内容”中看到可见性状态
- [ ] 若管理员填写了审核备注，作者能明显看到 `review_note`
- [ ] 驳回后的提示文案足够清楚

---

## 二、管理员路径

### 1. 默认工作台体验

- [ ] 进入后台内容页时默认优先查看 `待审核`
- [ ] 页面顶部待审核数量显示正确
- [ ] 作者筛选、分类筛选、关键词筛选可正常组合使用
- [ ] 清空筛选后能回到默认工作模式

### 2. 单条审核

- [ ] 操作列里的 `审核` 能正常打开弹框
- [ ] 弹框标题、说明文案和当前状态显示正确
- [ ] 审核状态和可见性下拉值显示的是 label，而不是原始值
- [ ] 审核备注输入区域可正常编辑

### 3. 快捷预设理解度

- [ ] `公开 / 仅链接可见 / 待审 / 下线` 这些快捷预设容易理解
- [ ] tooltip 能说明每个预设的结果
- [ ] 常驻说明文案能解释“快捷预设”的用途
- [ ] 变更预览能及时反映当前选择

### 4. 批量审核

- [ ] 可多选内容
- [ ] 选中后批量操作条会出现
- [ ] 批量审核弹框数量显示正确
- [ ] 批量操作不会造成明显歧义或误操作风险

### 5. 审核历史

- [ ] 审核弹框中能看到“最近审核记录”区块
- [ ] 无历史时空态文案正常
- [ ] 有历史时能看到审核人、时间、状态变化、可见性变化
- [ ] 审核备注在历史记录里可见

### 6. 体验细节

- [ ] 操作列不会过度拥挤
- [ ] 审核弹框层级清晰，不显得信息堆积
- [ ] 重要状态颜色能帮助快速判断
- [ ] 文案整体统一，没有前后台叫法不一致的问题

---

## 三、公共可见性路径

### 1. 公共列表

- [ ] `/contents` 只展示 `已发布 + 已通过 + 公开` 内容
- [ ] 待审核内容不会出现在公共列表
- [ ] 私有内容不会出现在公共列表
- [ ] 已下线内容不会出现在公共列表

### 2. 用户主页

- [ ] 用户主页内容列表只展示对外公开内容
- [ ] `仅链接可见` 内容不会出现在用户主页列表
- [ ] 作者自己公开的内容不会被错误过滤掉

### 3. 公共详情

- [ ] `公开` 内容可以正常访问详情
- [ ] `仅链接可见` 内容可通过直达链接访问详情
- [ ] `待审核 / 私有 / 已下线` 内容对公共访客不可访问

### 4. 公开衍生入口

- [ ] 收藏公开页不会漏出不该公开的内容
- [ ] 点赞、浏览、评论等入口不会绕过可见性规则
- [ ] 分类统计只计算公开内容
- [ ] 用户主页统计与实际公开列表保持一致

---

## 四、边界与回归

### 1. 访问者身份差异

- [ ] 未登录用户访问规则正确
- [ ] 作者访问自己的未公开内容规则正确
- [ ] 管理员访问全部内容规则正确

### 2. 路径一致性

- [ ] `我的内容`、后台列表、公共列表的状态解释一致
- [ ] 同一内容在不同页面中的状态 badge 一致
- [ ] `仅链接可见`、`私有`、`已下线` 这些术语含义一致

### 3. 常见失败场景

- [ ] 不会出现列表隐藏了，但详情还能公共访问的情况
- [ ] 不会出现详情隐藏了，但统计/收藏/分类仍然漏出的情况
- [ ] 不会出现作者自己的公开内容在主页里消失的情况
- [ ] 不会出现后台筛选条件导致空白页或错误状态残留
